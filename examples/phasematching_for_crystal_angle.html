Phasematching for crystal angle¶

The goal is to plot how the count rate changes with respect to a change in alignment (in theta) between the signal and its fiber coupling.

Let’s start with a yaml configuration (which could have been a file)

[1]:

import numpy as np
import math
import spectra
import os
import yaml
import plotly.graph_objects as go
from spdcalc import *

# radians to degrees
deg = math.pi / 180

YAML_CONFIG = """
apodization_enabled: false
crystal: KTP
crystal_length: 2000.0e-6
crystal_phi: 0.0
crystal_temperature: 293.15
crystal_theta: 1.5707963267948966
fiber_coupling: true
pm_type: Type2_e_eo
periodic_poling_enabled: false
pump_average_power: 0.001
pump_bandwidth: 5.35e-09
pump_spectrum_threshold: 1.0e-09
pump_waist: 100.0e-06
pump_wavelength: 775.0e-9
signal_phi: 0.0
signal_theta: 0.0
signal_theta_external: 0.0
signal_waist: 100.0e-06
signal_wavelength: 1550.0e-09
"""

Load the settings into a SPDCSetup object

[2]:

setup = SPDCSetup.from_dict(yaml.load(YAML_CONFIG, Loader=yaml.FullLoader))
# use periodic poling... the optimum
setup.assign_optimum_periodic_poling()

Setup a color scale in L.A.B. color space that we can pass to plotly

[3]:

colors = spectra.scale([ 'white', '#2c3e50' ]).colorspace('lab')
colorscale = [[z/1000, colors(z/1000).hexcode] for z in range(0, 1001)]

The Steps2D iterator makes it very easy to iterate over a 2D grid. We setup a grid with \(x \in [80, 100]\) in 100 steps, and \(y \in [0, 90]\) in 100 steps.

[4]:

ranges = Steps2D(
    (80, 100, 100),
    (0, 90, 100)
)

Setup a helper to calculate the value of the JSI at a given crystal orientation (\(\phi, \theta\))

[5]:

def jsi_for_theta_phi(setup, phi, theta):
    crystal_setup = setup.get_crystal_setup()
    crystal_setup.theta = theta * deg
    crystal_setup.phi = phi * deg
    setup.set_crystal_setup(crystal_setup)
    return abs(phasematch.phasematch_coincidences(setup)) ** 2

Calculate the normalization for the JSI which is the value at \((\phi, \theta) = (0^{\circ}, 90^{\circ})\). Then iterate over the grid and get the normalized JSI for each data point.

[6]:

norm = abs(jsi_for_theta_phi(setup, 0, 90)) ** 2
data = [ jsi_for_theta_phi(setup, phi, theta)/norm for (theta, phi) in ranges ]

Plot the data…

[8]:

# plot the data in plotly
fig = go.Figure(data=go.Heatmap(
    z=np.reshape(data, (100, 100)),
    x=ranges.get_x_values(),
    y=ranges.get_y_values(),
    colorscale=colorscale)
)

fig.update_layout(
    title='JSI for crystal orientation'
)

fig.show()
